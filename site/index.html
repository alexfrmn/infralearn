Зачем свой сервер?

AI-ассистенты вроде Claude Code и Codex работают в терминале. Чтобы получить от них максимум, нужен сервер, который работает 24/7. Вот три главные причины:

  • 24/7 доступ — сервер не спит, не перезагружается, не разряжается. Вы можете подключиться с любого устройства в любой момент.
  • Память и контекст — все ваши настройки, скиллы, MCP-серверы и история сохраняются. Каждая новая сессия начинается с того места, где вы остановились.
  • Приватность — ваши файлы, ключи API и проекты хранятся на вашем сервере, а не на чужом компьютере.

Почему сервер, а не ноутбук?

Вариант 1: Claude Code на ноутбуке (опасно через VPN)

Вариант 2: Claude Code на сервере (правильно)

Что мы настроим за день

  1. Арендуем VPS и подключимся через Termius
  2. Установим Node.js, Git и базовые утилиты
  3. Установим и настроим Claude Code
  4. Создадим CLAUDE.md и подключим скиллы
  5. Настроим память и MCP-серверы
  6. Настроим хуки
  7. Установим Codex CLI и сравним с Claude Code
  8. Бонус: OpenClaw, VPN

Шаг 2. Подключение через Termius

Устанавливаем Termius

Скачайте Termius с termius.com. Доступен для macOS, Windows, Linux, iOS и Android. Бесплатной версии достаточно.

Создаём новый хост

В Termius нажмите + New Host и заполните:

  • Label: AI Server (или любое название)
  • Hostname: ваш IP-адрес
  • Port: 22
  • Username: root
  • Password: ваш пароль из письма

Нажмите Save, затем дважды кликните на хост для подключения.

Первое подключение

При первом подключении Termius спросит, доверяете ли вы серверу (fingerprint). Нажмите Yes / Accept.

Если видите приглашение root@server:~# — вы подключены!

Проверка подключения

Выполните команду для проверки:

bash 
whoami && hostname && uptime

Должны увидеть root, имя сервера и время работы.

Не подключается?

Проверьте: правильный ли IP, порт 22, пароль. Если хостинг только создал VPS — подождите 2-3 минуты, сервер может ещё загружаться. Подробнее — на вкладке Проблемы.

Tailscale — приватная сеть между устройствами

Tailscale создаёт защищённую сеть между вашим ноутбуком, телефоном и сервером. После настройки устройства видят друг друга по приватным IP — как в одной локальной сети. Бесплатно до 100 устройств.

Зачем Tailscale?
  • Доступ к серверу по приватному IP (не нужно помнить публичный)
  • Шифрованное соединение (WireGuard под капотом)
  • Работает через NAT и файрволы — подключение из любой сети
  • SFTP из Termius на сервер по приватному адресу
  • Доступ с телефона к серверу

Шаг 1: Регистрация

  1. Зайдите на login.tailscale.com/start
  2. Зарегистрируйтесь через Google, GitHub или email
  3. Бесплатный план — Personal, до 100 устройств

Шаг 2: Установка на сервер

bash — на сервере (через Termius)

curl -fsSL https://tailscale.com/install.sh | sh

# Запуск и авторизация

Команда tailscale up покажет ссылку — откройте её в браузере на ноутбуке и авторизуйтесь. Сервер появится в вашей сети Tailscale.

bash 
# Проверяем — должен показать приватный IP (100.x.x.x)
tailscale ip -4
tailscale status

Шаг 5: SFTP в Termius

Termius умеет не только SSH, но и SFTP — передача файлов мышкой:

  1. Подключитесь к серверу в Termius
  2. Нажмите иконку SFTP (папка со стрелкой) в панели слева
  3. Слева — файлы ноутбука, справа — файлы сервера

tmux — ваша страховка от обрывов связи

Проблема без tmux

Вы работаете через Termius. WiFi моргнул, ноутбук заснул, Termius закрылся — и всё, что работало на сервере, умирает. Claude Code, Codex, скрипты — всё прерывается. Приходится начинать заново.

Решение: tmux

tmux — терминальный мультиплексор. Он создаёт сессию на сервере, которая живёт независимо от вашего подключения. Отключились → подключились обратно → tmux attach → всё на месте, как будто никуда не уходили.

Основные команды

bash 
# Создать новую сессию (делайте это СРАЗУ после подключения!)
tmux

# Всё! Теперь вы внутри tmux. Работайте как обычно.
# Claude Code, Codex, скрипты — всё будет жить даже если отключитесь.

Отключились? Не паника!

bash 
# Подключаемся обратно к серверу через Termius, затем:
tmux attach

# Если несколько сессий — посмотреть список:
tmux ls

# Подключиться к конкретной:
tmux attach -t 0

Горячие клавиши внутри tmux

Все команды tmux начинаются с Ctrl+B (нажать и отпустить), затем клавиша:

КомбинацияДействиеКогда нужно
Ctrl+B, затем cНовое окноХотите второй терминал (Claude + Codex одновременно)
Ctrl+B, затем nСледующее окноПереключиться между окнами
Ctrl+B, затем pПредыдущее окноПереключиться назад
Ctrl+B, затем %Разделить вертикальноДва терминала рядом
Ctrl+B, затем "Разделить горизонтальноДва терминала сверху/снизу
Ctrl+B, затем стрелкиПеремещение между панелямиПереключаться в split-режиме
Ctrl+B, затем dОтключиться (detach)Уйти, но оставить сессию живой
Ctrl+B, затем [Режим прокруткиПрокрутить вверх (выход — q)

Типичный рабочий день

bash 
# Утро: подключился через Termius
tmux                         # или tmux attach если сессия уже есть

# Окно 1: Claude Code
cd /opt/workspace && claude

# Ctrl+B, c — новое окно
# Окно 2: Codex
cd /opt/workspace && codex

# Ctrl+B, n / p — переключаемся между Claude и Codex
# Ctrl+B, d — отсоединяемся (сессии живут!)

# Вечер: подключился с телефона через Termius Mobile
tmux attach                  # всё на месте!

Именованные сессии (продвинуто)

bash 
# Создать сессию с именем
tmux new -s work

# Подключиться к именованной сессии
tmux attach -t work

# Список всех сессий
tmux ls
# work: 2 windows (created Sat Mar 21 14:30:00 2026)

# Убить сессию (когда больше не нужна)
tmux kill-session -t work
Золотое правило

Подключился к серверу → сразу tmux (или tmux attach если сессия уже была). Это должно стать привычкой. Без tmux любой обрыв связи = потеря работы.

Блок 5 — Memory + MCP

40 + 50 мин

Часть 1: Память — два уровня

По умолчанию Claude Code забывает всё после закрытия сессии. Это главная проблема. Решаем её двумя механизмами:

Два уровня памяти
  • MEMORY.md — встроенная авто-память. Claude сам записывает важное в файл ~/.claude/MEMORY.md (до 200 строк). Работает из коробки, ничего ставить не надо.
  • claude-mem (MCP) — продвинутая память. Граф знаний без лимита: сущности, факты, связи. Нужно установить MCP-сервер.

Уровень 1: MEMORY.md (уже работает!)

Claude Code автоматически создаёт файл ~/.claude/MEMORY.md и записывает туда важные вещи из разговора. Проверьте:

bash 
# Посмотреть что Claude уже запомнил
cat ~/.claude/MEMORY.md

Вы можете сами попросить Claude что-то запомнить — он добавит это в MEMORY.md:

# Внутри Claude Code скажите:
Запомни: мой основной язык — Python, я работаю над проектом X

Уровень 2: claude-mem (MCP-сервер)

Для серьёзной работы нужен граф знаний — без ограничения в 200 строк. Устанавливаем:

bash 
# Устанавливаем MCP-сервер памяти
claude mcp add memory -- npx -y @modelcontextprotocol/server-memory
Важно: нужна НОВАЯ сессия!

MCP-серверы загружаются при старте сессии. После claude mcp add нужно выйти из Claude Code (/exit) и зайти снова (claude).

После установки Claude Code получает инструменты для работы с памятью:

ИнструментЧто делаетПример
create_entitiesСоздать сущностиВы, проекты, серверы, технологии
add_observationsДобавить факты"Python 3.12", "использует PostgreSQL"
search_nodesПоиск по памятиНайти всё про "Python"
create_relationsСвязать сущности"Проект X" → использует → "PostgreSQL"

# 2. Говорим:

#    /exit

claude

# 5. Спрашиваем:
#    "Какой мой любимый язык? Где я работаю?"

# Claude должен ответить правильно из памяти!
Как это работает

MEMORY.md — простой текстовый файл, Claude читает его при каждом старте. Подходит для базовых фактов.
claude-mem — граф знаний в JSON. Claude может искать, добавлять и связывать факты. Подходит для сложного контекста: проекты, решения, конфигурации, люди.

Часть 2: MCP-серверы

MCP (Model Context Protocol) — это способ дать Claude Code доступ к внешним инструментам: файловой системе, GitHub, документации и другим сервисам.

Файловая система

bash 
claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /opt/workspace

Даёт Claude Code прямой доступ к файлам в /opt/workspace: чтение, запись, поиск, создание директорий.

GitHub

bash 
# Замените ghp_YOUR_TOKEN на ваш Personal Access Token из GitHub
claude mcp add github -e GITHUB_PERSONAL_ACCESS_TOKEN=ghp_YOUR_TOKEN \
  -- npx -y @modelcontextprotocol/server-github
Как получить GitHub Token

GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token. Дайте права: repo, read:org.

Context7 — Документация библиотек

bash 
claude mcp add context7 -- npx -y @upstash/context7-mcp@latest

Даёт Claude Code доступ к актуальной документации Python, Node.js и других библиотек. Больше никаких устаревших ответов.

Exa — Веб-поиск

Нужен API ключ Exa

Зарегистрируйтесь на exa.ai и получите API ключ (есть бесплатный план).

bash 
# Замените YOUR_EXA_API_KEY на ваш ключ!
claude mcp add exa -e EXA_API_KEY=YOUR_EXA_API_KEY \
  -- npx -y exa-mcp-server

Exa даёт Claude Code возможность искать актуальную информацию в интернете. Используйте скилл /exa-web-research для структурированных исследований.

Проверяем установленные MCP

Должны увидеть список из 5 серверов: memory, filesystem, github, context7, exa.

macOS: Obsidian Git не может пушить (SSH ошибка)

На macOS Obsidian — sandboxed приложение. Плагин Obsidian Git не имеет доступа к SSH-ключам (~/.ssh/) и не может пушить через SSH. Ошибка: git@github.com: Permission denied.

Решение — переключить remote на HTTPS:

bash — macOS Terminal 
# Перейти в папку vault
cd ~/my-vault

# Проверить текущий remote (если SSH — нужно менять)
git remote -v
# Если видите: git@github.com:USER/my-vault.git — это SSH, не будет работать

# Переключить на HTTPS:
git remote set-url origin https://github.com/ВАШ_USERNAME/my-vault.git

# Проверить
git remote -v
# Должно быть: https://github.com/ВАШ_USERNAME/my-vault.git

# Сохранить пароль (PAT) чтобы не вводить каждый раз
git config --global credential.helper osxkeychain
После этого

Obsidian Git сможет пушить. При первом push введите GitHub username и Personal Access Token вместо пароля. macOS Keychain запомнит — больше вводить не придётся.

Блок 6 — Хуки

30 мин

Что такое хуки?

Хуки — это скрипты, которые Claude Code автоматически запускает при определённых событиях. Например:

  • PreToolUse — перед выполнением инструмента (можно блокировать опасные операции)
  • PostToolUse — после выполнения инструмента
  • Notification — когда Claude хочет привлечь ваше внимание

Шаг 1. Копируем хуки из репозитория

bash 
mkdir -p ~/.claude/hooks
cp /opt/workspace/infralearn/hooks/*.sh ~/.claude/hooks/
chmod +x ~/.claude/hooks/*.sh
Важно: абсолютные пути

В настройках хуков всегда используйте абсолютные пути (/root/.claude/hooks/script.sh), а не относительные (~/.claude/hooks/script.sh). Относительные пути ломаются при работе из разных директорий.

Шаг 2. Настраиваем settings.local.json

bash 
cp /opt/workspace/infralearn/configs/settings.local.json ~/.claude/settings.local.json

Файл settings.local.json описывает, какие хуки запускать и когда. Вот что внутри:

Важно: только абсолютные пути!

В хуках ВСЕГДА используйте /root/.claude/hooks/, а НЕ ~/.claude/hooks/. Тильда ломается при работе из разных директорий.

json 
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "bash /root/.claude/hooks/session-start.sh"
          }
        ]
      }
    ],
    "SessionEnd": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "bash /root/.claude/hooks/session-end.sh"
          }
        ]
      }
    ]
  }
}
Структура хука

Каждый хук: событие (SessionStart, SessionEnd, PreToolUse...) → matcher (фильтр, пустая строка = все) → hooks массив с командами. Обязательно вложенная структура с matcher + hooks!

Шаг 3. Проверяем

bash 
# Проверяем, что файлы на месте
ls -la ~/.claude/hooks/
cat ~/.claude/settings.local.json
Совет

Начните с простого хука уведомлений (отправка в Telegram, когда Claude завершил долгую задачу). Сложные хуки с блокировкой добавляйте позже, когда поймёте, какие операции хотите контролировать.

Пример: хук уведомления

bash 
cat > ~/.claude/hooks/notify.sh << 'HOOKEOF'
#!/bin/bash
# Простой хук: логирует уведомления Claude Code
LOGFILE="/tmp/claude-notifications.log"
INPUT=$(cat)
TITLE=$(echo "$INPUT" | jq -r '.title // "no title"')
MSG=$(echo "$INPUT" | jq -r '.message // "no message"')
echo "[$(date '+%Y-%m-%d %H:%M:%S')] $TITLE: $MSG" >> "$LOGFILE"
HOOKEOF
chmod +x ~/.claude/hooks/notify.sh

Statusline — строка внизу Claude Code, которая показывает полезную информацию: модель, контекст, время сессии, расход.

Как выглядит

  • Контекст — сколько % окна контекста использовано (прогресс-бар)

  • Изменения — добавленные/удалённые строки кода

Решение проблем

справочник
SSH не подключается (Connection refused / timeout)
  • Проверьте IP-адрес — опечатка?
  • Порт должен быть 22 (стандартный SSH)
  • VPS только что создан? Подождите 2-3 минуты
  • Проверьте, что VPS запущен в панели хостинга
  • Попробуйте пинг: если хостинг даёт веб-консоль — проверьте через неё
Claude Code OAuth — не открывается URL
  • URL нужно открыть на ноутбуке в браузере, не на сервере
  • Скопируйте URL полностью (с https://)
  • Убедитесь, что вы залогинены на console.anthropic.com
  • Проверьте, что подписка (Pro или Max) активна
MCP timeout / не запускается
  • npx скачивает пакет при каждом запуске, что медленно. Установите глобально:
bash 
# Вместо npx — глобальная установка
npm install -g @modelcontextprotocol/server-memory
npm install -g @modelcontextprotocol/server-filesystem
npm install -g @modelcontextprotocol/server-github

Затем в claude mcp add используйте прямой путь к бинарнику вместо npx.

Permission denied (нет прав)
  • Для скриптов: chmod +x filename.sh
  • Для директорий: chmod 755 /path/to/dir
  • Внутри Claude Code: используйте команду /permissions для управления разрешениями
  • Вы работаете под root? Проверьте: whoami
Claude Code не отвечает / зависает
  • Выполните /status — проверьте статус подключения
  • Проверьте подписку на console.anthropic.com
  • Попробуйте /clear и начните новый диалог
  • Выйдите (Ctrl+C) и запустите заново: claude
  • Проверьте интернет на сервере: curl -s https://api.anthropic.com
Codex CLI не авторизуется
  • Попробуйте OAuth через браузер (как с Claude Code)
  • Если OAuth не работает — используйте API Key:
bash 
# Установите API ключ OpenAI
export OPENAI_API_KEY="sk-your-key-here"
codex

Ключ можно получить на platform.openai.com/api-keys.

Node.js не найден после установки
  • Повторите установку:
bash 
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs
node -v
  • Если ошибка остаётся — проверьте PATH: echo $PATH
  • Node.js должен быть в /usr/bin/node: which node
npm WARN / npm ERR при установке пакетов
  • WARN — обычно можно игнорировать, это предупреждения
  • ERR — ошибка. Частые причины:
    • Нет интернета — проверьте: ping google.com
    • Старая версия npm — обновите: npm install -g npm@latest
    • Нет места на диске — проверьте: df -h
Потерял соединение — работа пропала
  • Используйте tmux! Запускайте tmux сразу после подключения.
  • Если отключились, подключитесь снова и выполните:
bash 
tmux attach

Все процессы внутри tmux продолжают работать после отключения.

Вопросы, которые задавали чаще всего

Cursor vs CLI

«Почему терминал, а не Cursor?» — CLI даёт больше контроля, работает по SSH из любого места, память персистентна, можно автоматизировать через хуки и скиллы. Claude Code в терминале «гораздо мощнее, чем в приложении — можно затюнить в бешеную собаку на стероидах».

Claude vs Codex

«Зачем оба?» — Claude (Anthropic) для архитектуры и сложных задач, Codex (OpenAI) для кодинга. Разные модели, разные сильные стороны. Через Murmur можно заставить их работать вместе.

Max vs Pro

«В чём разница?» — только в лимитах. На Pro недельный лимит достигался 2 раза. Max = безлимит для серьёзной работы.

VPN на сервере

«Если выключить VPN на компе, Claude продолжит работать?» — Да! Сервер работает 24/7. VPN нужен только вашему ноутбуку. Claude на сервере не зависит от вашего VPN.
