Зачем свой сервер?

AI-ассистенты вроде Claude Code и Codex работают в терминале. Чтобы получить от них максимум, нужен сервер, который работает 24/7. Вот три главные причины:

  • 24/7 доступ — сервер не спит, не перезагружается, не разряжается. Вы можете подключиться с любого устройства в любой момент.
  • Память и контекст — все ваши настройки, скиллы, MCP-серверы и история сохраняются. Каждая новая сессия начинается с того места, где вы остановились.
  • Приватность — ваши файлы, ключи API и проекты хранятся на вашем сервере, а не на чужом компьютере.

Почему сервер, а не ноутбук?

Вариант 1: Claude Code на ноутбуке (опасно через VPN)

Вариант 2: Claude Code на сервере (правильно)

Что мы настроим за день

  1. Арендуем VPS и подключимся через Termius
  2. Установим Node.js, Git и базовые утилиты
  3. Установим и настроим Claude Code
  4. Создадим CLAUDE.md и подключим скиллы
  5. Настроим память и MCP-серверы
  6. Настроим хуки
  7. Установим Codex CLI и сравним с Claude Code
  8. Бонус: OpenClaw, VPN
Совет

Не нужно всё запоминать. В конце курса есть шпаргалка со всеми командами. Главное — понять логику и попробовать руками.

Что понадобится

  • Ноутбук с браузером (для OAuth-авторизации)
  • Termius (бесплатный SSH-клиент, скачайте заранее на termius.com)
  • Подписка Claude Pro или Max (для Claude Code)
  • Подписка ChatGPT Plus (для Codex CLI) — опционально
  • Банковская карта для оплаты VPS (~1540 руб/мес)

Блок 1 — VPS + Подключение

40 мин

Шаг 1. Покупка VPS

Открываем 4vps.su

Переходим на сайт 4vps.su и выбираем раздел VPS/VDS.

Выбираем тариф

Нам нужен тариф USA-cx31 (или аналогичный) со следующими параметрами:

ПараметрЗначение
CPU4 ядра
RAM4 GB
Диск40 GB NVMe
ЛокацияUSA
Цена~1540 руб/мес
Важно: локация

Выбирайте сервер в USA. Claude Code и Codex API работают из США, и пинг будет минимальным. Российские серверы тоже подойдут, но API-запросы будут идти дольше.

Выбираем ОС

Операционная система: Ubuntu 24.04 LTS. Это стабильный и долгосрочный релиз, все команды в курсе написаны под него.

Оплата и получение доступа

После оплаты вам придёт письмо с данными для подключения:

  • IP-адрес сервера (например, 203.0.113.42)
  • Логин: root
  • Пароль

Шаг 2. Подключение через Termius

Устанавливаем Termius

Скачайте Termius с termius.com. Доступен для macOS, Windows, Linux, iOS и Android. Бесплатной версии достаточно.

Создаём новый хост

В Termius нажмите + New Host и заполните:

  • Label: AI Server (или любое название)
  • Hostname: ваш IP-адрес
  • Port: 22
  • Username: root
  • Password: ваш пароль из письма

Нажмите Save, затем дважды кликните на хост для подключения.

Первое подключение

При первом подключении Termius спросит, доверяете ли вы серверу (fingerprint). Нажмите Yes / Accept.

Если видите приглашение root@server:~# — вы подключены!

Проверка подключения

Выполните команду для проверки:

bash 
whoami && hostname && uptime

Должны увидеть root, имя сервера и время работы.

Не подключается?

Проверьте: правильный ли IP, порт 22, пароль. Если хостинг только создал VPS — подождите 2-3 минуты, сервер может ещё загружаться. Подробнее — на вкладке Проблемы.

Блок 2 — Настройка сервера

40 мин

Шаг 1. Обновляем систему

Первым делом обновим все пакеты до актуальных версий:

bash 
apt update && apt upgrade -y
Что происходит

apt update — обновляет список доступных пакетов. apt upgrade -y — устанавливает обновления без вопросов (-y = автоматическое «да»).

Шаг 2. Устанавливаем базовые утилиты

bash 
apt install -y git curl wget htop tmux jq unzip

Что устанавливаем:

  • git — система контроля версий
  • curl, wget — скачивание файлов из интернета
  • htop — мониторинг ресурсов сервера
  • tmux — мультиплексор терминала (несколько окон в одном)
  • jq — работа с JSON в командной строке
  • unzip — распаковка архивов

Шаг 3. Устанавливаем Node.js 22

Claude Code и Codex CLI написаны на Node.js. Нам нужна версия 22 (LTS):

bash 
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs

Шаг 4. Проверяем версии

bash 
node -v && npm -v && python3 --version && git --version

Ожидаемый результат (версии могут отличаться):

v22.x.x
10.x.x
Python 3.12.x
git version 2.x.x
Важно

Если node -v показывает ошибку — повторите шаг 3. Если python3 не найден — на Ubuntu 24.04 он должен быть предустановлен. Установите вручную: apt install -y python3.

Шаг 5. Настраиваем Git

bash 
git config --global user.name "Your Name"
git config --global user.email "your@email.com"

Замените Your Name и your@email.com на ваши реальные данные. Это нужно для коммитов.

Шаг 6. Создаём рабочую директорию

bash 
mkdir -p /opt/workspace && cd /opt/workspace

Все наши проекты будут в /opt/workspace/. Это стандартная практика для серверов.

Совет: tmux

Если потеряете соединение с сервером, вся работа в обычном терминале пропадёт. Используйте tmux — он сохраняет сессию. Запустите tmux перед началом работы, и если отключитесь — подключитесь снова и введите tmux attach.

Блок 3 — Claude Code

50 мин

Шаг 1. Установка

bash 
npm install -g @anthropic-ai/claude-code

Флаг -g устанавливает глобально — команда claude будет доступна из любой директории.

Шаг 2. Первый запуск

bash 
cd /opt/workspace

Шаг 3. OAuth-авторизация

При первом запуске Claude Code покажет URL для авторизации. Вот что делать:

  1. Скопируйте URL из терминала (он начинается с https://console.anthropic.com/...)
  2. Откройте его в браузере на ноутбуке (не на сервере!)
  3. Войдите в свой аккаунт Anthropic
  4. Подтвердите авторизацию
  5. Скопируйте код и вставьте его обратно в терминал
Важно

URL нужно открыть именно на ноутбуке (в обычном браузере), а не на сервере. На сервере нет браузера. Код из URL вставьте обратно в терминал сервера.

Шаг 4. Первый диалог

Попробуем дать Claude Code первую задачу. Введите прямо в чате Claude:

Claude Code создаст Python-файл, объяснит его и предложит запустить. Разрешите выполнение и посмотрите результат.

Блок 4 — CLAUDE.md + Skills

40 + 40 мин

Часть 1: CLAUDE.md

CLAUDE.md — это файл инструкций для Claude Code. Он определяет, как AI будет себя вести, какие правила соблюдать, какой контекст учитывать.

Два уровня CLAUDE.md

УровеньПутьОписание
Глобальный~/.claude/CLAUDE.mdДействует во всех проектах
Проектный./CLAUDE.mdДействует только в текущем проекте

Проектный CLAUDE.md имеет приоритет и дополняет глобальный.

Скачиваем шаблоны

bash 
# Клонируем репозиторий курса
git clone https://github.com/alexfrmn/infralearn.git /opt/workspace/infralearn
bash 
# Устанавливаем глобальный CLAUDE.md
mkdir -p ~/.claude
cp /opt/workspace/infralearn/configs/CLAUDE.md ~/.claude/CLAUDE.md
Что внутри шаблона

Шаблон содержит базовые правила: язык ответов, стиль кода, правила безопасности. Вы можете (и должны!) адаптировать его под себя.

Что писать в CLAUDE.md

  • Кто вы — ваша роль, стек технологий, уровень
  • Правила — язык, стиль кода, что можно/нельзя
  • Контекст проекта — архитектура, ключевые файлы
  • Память — что важно помнить между сессиями

Часть 2: Skills (Скиллы)

Скиллы — это готовые наборы инструкций для типовых задач. Вместо того чтобы каждый раз объяснять Claude, как деплоить, коммитить или писать тесты, вы вызываете скилл одной командой.

Устанавливаем скиллы

bash 
# Копируем скиллы из репозитория курса
mkdir -p ~/.claude/skills
cp -r /opt/workspace/infralearn/skills/* ~/.claude/skills/
bash 
# Проверяем, что скиллы на месте
ls ~/.claude/skills/

Как использовать скиллы

Внутри Claude Code введите / и начните набирать название скилла. Claude покажет список доступных. Например:

  • /commit — создать осмысленный коммит
  • /review — ревью кода
  • /deploy — деплой проекта

Создаём свой скилл

Скилл — это обычный Markdown-файл. Создадим простой:

bash 
cat > ~/.claude/skills/hello.md << 'EOF'
---
name: hello
description: Приветствие и статус сервера
---

Покажи:
1. Текущее время на сервере
2. Uptime
3. Загрузку CPU и памяти (htop-style)
4. Свободное место на диске

Формат: компактный, в одном сообщении.
EOF

Теперь в Claude Code можно вызвать /hello, и он покажет статус сервера.

Совет

Начните с 3-5 скиллов для повседневных задач. Со временем вы накопите библиотеку, которая сэкономит часы.

Блок 5 — Memory + MCP

40 + 50 мин

Часть 1: Память (Memory)

По умолчанию Claude Code не помнит предыдущие сессии. MCP-сервер Memory решает эту проблему — он сохраняет факты между сессиями в виде графа знаний.

Установка Memory

bash 
claude mcp add memory -- npx -y @modelcontextprotocol/server-memory

После установки Claude Code получает инструменты для работы с памятью:

  • create_entities — создать сущности (вы, проекты, серверы)
  • add_observations — добавить факты к сущностям
  • search_nodes — поиск по памяти
  • create_relations — связи между сущностями
Как использовать

Просто скажите Claude: «Запомни, что мой основной проект — на Python 3.12, база — PostgreSQL». Он сохранит это в память и вспомнит в следующей сессии.

Часть 2: MCP-серверы

MCP (Model Context Protocol) — это способ дать Claude Code доступ к внешним инструментам: файловой системе, GitHub, документации и другим сервисам.

Файловая система

bash 
claude mcp add filesystem -- npx -y @modelcontextprotocol/server-filesystem /opt/workspace

Даёт Claude Code прямой доступ к файлам в /opt/workspace: чтение, запись, поиск, создание директорий.

GitHub

bash 
# Замените ghp_YOUR_TOKEN на ваш Personal Access Token из GitHub
claude mcp add github -e GITHUB_PERSONAL_ACCESS_TOKEN=ghp_YOUR_TOKEN \
  -- npx -y @modelcontextprotocol/server-github
Как получить GitHub Token

GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token. Дайте права: repo, read:org.

Context7 — Документация библиотек

bash 
claude mcp add context7 -- npx -y @upstash/context7-mcp@latest

Даёт Claude Code доступ к актуальной документации Python, Node.js и других библиотек. Больше никаких устаревших ответов.

Exa — Веб-поиск

Нужен API ключ Exa

Зарегистрируйтесь на exa.ai и получите API ключ (есть бесплатный план).

bash 
# Замените YOUR_EXA_API_KEY на ваш ключ!
claude mcp add exa -e EXA_API_KEY=YOUR_EXA_API_KEY \
  -- npx -y exa-mcp-server

Exa даёт Claude Code возможность искать актуальную информацию в интернете. Используйте скилл /exa-web-research для структурированных исследований.

Проверяем установленные MCP

bash 
claude mcp list

Должны увидеть список из 5 серверов: memory, filesystem, github, context7, exa.

Часть 3: Obsidian Vault на сервере

Создадим базу знаний (vault) прямо на сервере. Это удобно для хранения заметок, документации и всего, что Claude должен помнить.

bash 
mkdir -p /opt/workspace/vault/{00-Inbox,Projects,Daily-Notes}
cd /opt/workspace/vault
git init
echo "# My Knowledge Base" > README.md
git add . && git commit -m "init vault"
Зачем vault на сервере?

Claude Code может читать и писать в vault через MCP filesystem. Это значит, что он может сам создавать заметки, искать по ним и использовать как долгосрочную память. Если вы используете Obsidian на компьютере — синхронизируйте через Git.

Блок 6 — Хуки

30 мин

Что такое хуки?

Хуки — это скрипты, которые Claude Code автоматически запускает при определённых событиях. Например:

  • PreToolUse — перед выполнением инструмента (можно блокировать опасные операции)
  • PostToolUse — после выполнения инструмента
  • Notification — когда Claude хочет привлечь ваше внимание

Шаг 1. Копируем хуки из репозитория

bash 
mkdir -p ~/.claude/hooks
cp /opt/workspace/infralearn/hooks/*.sh ~/.claude/hooks/
chmod +x ~/.claude/hooks/*.sh
Важно: абсолютные пути

В настройках хуков всегда используйте абсолютные пути (/root/.claude/hooks/script.sh), а не относительные (~/.claude/hooks/script.sh). Относительные пути ломаются при работе из разных директорий.

Шаг 2. Настраиваем settings.local.json

bash 
cp /opt/workspace/infralearn/configs/settings.local.json ~/.claude/settings.local.json

Файл settings.local.json описывает, какие хуки запускать и когда. Пример структуры:

json 
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "/root/.claude/hooks/pre-bash.sh"
          }
        ]
      }
    ],
    "Notification": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "/root/.claude/hooks/notify.sh"
          }
        ]
      }
    ]
  }
}

Шаг 3. Проверяем

bash 
# Проверяем, что файлы на месте
ls -la ~/.claude/hooks/
cat ~/.claude/settings.local.json
Совет

Начните с простого хука уведомлений (отправка в Telegram, когда Claude завершил долгую задачу). Сложные хуки с блокировкой добавляйте позже, когда поймёте, какие операции хотите контролировать.

Пример: хук уведомления

bash 
cat > ~/.claude/hooks/notify.sh << 'HOOKEOF'
#!/bin/bash
# Простой хук: логирует уведомления Claude Code
LOGFILE="/tmp/claude-notifications.log"
INPUT=$(cat)
TITLE=$(echo "$INPUT" | jq -r '.title // "no title"')
MSG=$(echo "$INPUT" | jq -r '.message // "no message"')
echo "[$(date '+%Y-%m-%d %H:%M:%S')] $TITLE: $MSG" >> "$LOGFILE"
HOOKEOF
chmod +x ~/.claude/hooks/notify.sh

Блок 8 — Бонус: OpenClaw / VPN

если успеем

OpenClaw — Claude в Telegram

OpenClaw позволяет общаться с Claude Code через Telegram-бота. Это удобно: пишете в Telegram с телефона, а Claude выполняет задачи на сервере.

Установка

bash 
npm install -g openclaw

Создание бота

  1. Откройте Telegram, найдите @BotFather
  2. Отправьте /newbot
  3. Придумайте имя и username для бота
  4. Скопируйте токен бота (формат: 1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ)

Настройка

bash 
# Инициализация — мастер настройки
openclaw init

Мастер спросит токен бота и другие параметры. После настройки:

bash 
# Запуск
openclaw start
Важно

OpenClaw требует активную подписку Claude. Убедитесь, что Claude Code работает на сервере перед настройкой OpenClaw.

Upload2Server — передача файлов через Telegram

Telegram-бот, который принимает файлы и сохраняет их на сервере. Удобно для скриншотов, документов, конфигов — кинул в чат, и оно на сервере.

Шаг 1: Создаём бота

  1. Откройте @BotFather в Telegram
  2. Отправьте /newbot
  3. Придумайте имя (например: "My Upload Bot")
  4. Скопируйте токен бота
  5. Узнайте свой Telegram ID: отправьте /start боту @userinfobot

Шаг 2: Устанавливаем бота

bash 
# Копируем бота из репо
cd /opt/workspace
cp -r infralearn/upload-bot .
cd upload-bot

# Создаём виртуальное окружение
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

Шаг 3: Настраиваем

bash 
# Создаём файл с настройками
cat > .env << EOF

VPN: OpenVPN за 10 минут

Раз у вас есть VPS в USA — можно поднять свой VPN. OpenVPN — проверенное решение, работает на всех платформах.

bash 
# Скачиваем установщик OpenVPN
curl -O https://raw.githubusercontent.com/angristan/openvpn-install/master/openvpn-install.sh
chmod +x openvpn-install.sh
sudo ./openvpn-install.sh
Что отвечать на вопросы установщика
  • IP address: оставьте по умолчанию (ваш IP)
  • IPv6: n
  • Port: 1194 (по умолчанию)
  • Protocol: UDP
  • DNS: 3 (Cloudflare)
  • Compression: n
  • Encryption: по умолчанию
  • Client name: phone (или laptop)

После установки файл конфигурации клиента появится в ~/phone.ovpn

Как подключиться

  1. Скачайте файл .ovpn на ноутбук/телефон (через Upload2Server бот или SFTP в Termius)
  2. Установите OpenVPN Connect (App Store / Google Play / Desktop)
  3. Импортируйте файл .ovpn
  4. Подключитесь!
bash 
# Добавить ещё клиента (laptop, tablet и т.д.)
sudo ./openvpn-install.sh

# Проверить статус OpenVPN
sudo systemctl status openvpn-server@server
Совет

Создайте отдельный клиент для каждого устройства (phone, laptop, tablet). Так можно отозвать доступ для конкретного устройства если потеряете его.

Решение проблем

справочник
SSH не подключается (Connection refused / timeout)
  • Проверьте IP-адрес — опечатка?
  • Порт должен быть 22 (стандартный SSH)
  • VPS только что создан? Подождите 2-3 минуты
  • Проверьте, что VPS запущен в панели хостинга
  • Попробуйте пинг: если хостинг даёт веб-консоль — проверьте через неё
Claude Code OAuth — не открывается URL
  • URL нужно открыть на ноутбуке в браузере, не на сервере
  • Скопируйте URL полностью (с https://)
  • Убедитесь, что вы залогинены на console.anthropic.com
  • Проверьте, что подписка (Pro или Max) активна
MCP timeout / не запускается
  • npx скачивает пакет при каждом запуске, что медленно. Установите глобально:
bash 
# Вместо npx — глобальная установка
npm install -g @modelcontextprotocol/server-memory
npm install -g @modelcontextprotocol/server-filesystem
npm install -g @modelcontextprotocol/server-github

Затем в claude mcp add используйте прямой путь к бинарнику вместо npx.

Permission denied (нет прав)
  • Для скриптов: chmod +x filename.sh
  • Для директорий: chmod 755 /path/to/dir
  • Внутри Claude Code: используйте команду /permissions для управления разрешениями
  • Вы работаете под root? Проверьте: whoami
Claude Code не отвечает / зависает
  • Выполните /status — проверьте статус подключения
  • Проверьте подписку на console.anthropic.com
  • Попробуйте /clear и начните новый диалог
  • Выйдите (Ctrl+C) и запустите заново: claude
  • Проверьте интернет на сервере: curl -s https://api.anthropic.com
Codex CLI не авторизуется
  • Попробуйте OAuth через браузер (как с Claude Code)
  • Если OAuth не работает — используйте API Key:
bash 
# Установите API ключ OpenAI
export OPENAI_API_KEY="sk-your-key-here"
codex

Ключ можно получить на platform.openai.com/api-keys.

Node.js не найден после установки
  • Повторите установку:
bash 
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs
node -v
  • Если ошибка остаётся — проверьте PATH: echo $PATH
  • Node.js должен быть в /usr/bin/node: which node
npm WARN / npm ERR при установке пакетов
  • WARN — обычно можно игнорировать, это предупреждения
  • ERR — ошибка. Частые причины:
    • Нет интернета — проверьте: ping google.com
    • Старая версия npm — обновите: npm install -g npm@latest
    • Нет места на диске — проверьте: df -h
Потерял соединение — работа пропала
  • Используйте tmux! Запускайте tmux сразу после подключения.
  • Если отключились, подключитесь снова и выполните:
bash 
tmux attach

Все процессы внутри tmux продолжают работать после отключения.
